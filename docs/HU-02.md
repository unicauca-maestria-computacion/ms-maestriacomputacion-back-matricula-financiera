# HU-02: Consultar estudiante individual por código

**As a** sistema frontend de gestión de maestría
**I want** obtener los datos financieros completos de un estudiante específico por su código
**So that** pueda mostrar su información de matrícula financiera individual con valor en SMLV

---

## Business Rules

- BR-01: El código del estudiante no puede ser nulo ni vacío.
- BR-02: El estudiante debe existir en la base de datos.
- BR-03: Si no se pasa periodo (tagPeriodo/año), se usa el periodo académico con estado ACTIVO.
- BR-04: El valor en SMLV se calcula con las mismas reglas de HU-01.
- BR-05: Los datos personales (nombre, apellido, identificación) se enriquecen desde la tabla `personas`.

---

## Acceptance Criteria (BDD)

**Scenario 1 — Happy Path: Estudiante existe con periodo activo**
- Given: estudiante con código `"EST001"` existe en BD y hay un periodo con estado ACTIVO
- When: `GET /api/v1/gestion-matricula-financiera/estudiantes/EST001`
- Then: HTTP 200 con `StudentResponse` completo incluyendo `valorEnSMLV`, nombre, apellido y materias del periodo activo

**Scenario 2 — Edge Case: Estudiante existe, se pasa periodo específico**
- Given: estudiante `"EST001"` existe, se pasa `?tagPeriodo=2&anio=2024`
- When: `GET /api/v1/gestion-matricula-financiera/estudiantes/EST001?tagPeriodo=2&anio=2024`
- Then: HTTP 200 con materias del periodo 2-2024 específicamente

**Scenario 3 — Excepción de negocio: Estudiante no existe**
- Given: código `"NOEXISTE"` no está en BD
- When: `GET /api/v1/gestion-matricula-financiera/estudiantes/NOEXISTE`
- Then: HTTP 404 con `ProblemDetail` donde `errorCode = "MF-0003"`

---

## Technical Specification

**Endpoint:** `GET /api/v1/gestion-matricula-financiera/estudiantes/{codigo}`
**Query params:** `tagPeriodo` (optional), `anio` (optional)
**Input port:** `ManageEnrolledStudentsUseCase.getStudentByCode(String, Integer, Integer)`
**Output port:** `StudentGatewayPort`
**Response DTO:** `StudentResponse`
**ErrorCodes:** MF-0003 (estudiante no encontrado), MF-0005 (código nulo)
**Security:** ROLE_MF_READ en prod, permitAll en dev
**BD tables:** `estudiantes`, `personas`, `matriculas`, `cursos`, `asignaturas`, `periodo_academico`
